THE BUG
================================================================================

Symptom: The bidding popup was appearing prematurely during Round 2, appearing
in the middle of card play instead of waiting for all 13 cards to finish.

Impact:
- Cards would disappear or become unclickable
- Game would jump between states unpredictably
- Round 2 could not be completed

Root Cause: State closure issue in React event handler

================================================================================
THE ROOT CAUSE (TECHNICAL EXPLANATION)
================================================================================

The code was checking a STALE CLOSURE VALUE instead of FRESH PAYLOAD DATA:

OLD (BROKEN):
    socket.on("start_game", (payload) => {
        if (state && state.round > 0) {  ← Uses old 'state' from closure
            applyServerStateNewHand(payload.initialGameState);
        }
    });

    Problem: When Round 2 event arrived, 'state' still had round=1 (stale)
    The popup was never closed with setBetPopupOpen(false)

NEW (FIXED):
    socket.on("start_game", (payload) => {
        if (payload.initialGameState.round > 1) {  ← Uses fresh payload data
            setBetPopupOpen(false);  ← Close previous popup
            applyServerStateNewHand(payload.initialGameState);
        }
    });

    Solution: Always use fresh data from payload, not stale state from closure
    Explicitly closes the old popup before showing new hand

================================================================================
KEY INSIGHT
================================================================================

This bug demonstrates a critical React principle:

"Never rely on stale closure state for control flow logic.
 Always use fresh data from props or event payloads when
 determining program behavior."

In this case:
- ❌ WRONG: Checking state.round (captured in closure, becomes stale)
- ✅ RIGHT: Checking payload.initialGameState.round (fresh from server)

This pattern should be remembered for future React development.
